# 問卷調查與使用者回饋指南 (Survey & User Feedback Guide)

本文件專為本專案未來發布問卷時提供參考，詳細規劃了**問卷發布的黃金時機**、**如何設計有價值的有效問題**，以及如何識別並避免那些**「問了等於白問」的陷阱問題**，確保回收的數據能對後續開發有實質指導意義。

---

## 📅 一、問卷發布的黃金時機 (When to Survey?)

問卷不是發得越頻繁越好，也不是開發全部結束才發。在不同開發階段，問卷的焦點應該完全不同：

### 1. 第一階段：封閉原型測試期 (Alpha / Prototype Test)
*   **時機點**：完成 **Phase 1 (客戶庫 CRUD)** 與 **Phase 5 (基礎日曆)** 之後。此時系統具備了最基本的客戶記錄與手動排程能力。
*   **對象**：3-5 位關係友好、願意提供真實回饋的保險業務代表。
*   **目的**：**易用性測試 (Usability)**。測試基本介面是否直觀、有無低級 bug、滑動順暢度。這個時期的問卷應以**任務導向**為主（例如：「您能在 1 分鐘內建立新客戶並設定明天提醒嗎？」）。

### 2. 第二階段：核心價值驗證期 (Beta Launch)
*   **時機點**：完成 **Phase 3 (語音錄音)** 與 **Phase 4 (Gemini 語意分析與重點提取)** 之後。此時本系統的「殺手級功能（AI 語音自動助理）」已可運作。
*   **對象**：擴大至 15-30 位業務代表。
*   **目的**：**價值主張驗證 (Value Proposition)**。驗證「語音轉錄＋AI 分析」是否真的有幫他們省下工作時間、AI 抓取的欄位是否夠精準。此時問卷重點在於**痛點解決程度**與**使用頻率**。

## 🎯 二、什麼是「有效問題」？(Effective Questions)

有效的問卷問題應該聚焦於**使用者的真實歷史行為**和**具體的情境痛點**，而不是猜想。

### 1. 行為情境題（探尋真實發生的行為）
*   **有效範例**：「在過去的一週中，您通常在什麼時候、什麼地點記錄客戶的訪談筆記？(複選)」
    *   *設計意圖*：這能幫我們了解使用者是在「行進中的馬路上」使用 App（這時用語音輸入非常重要），還是在「回到辦公室電腦前」使用（這時 Web 網頁版的鍵盤快速鍵和排版更重要）。
*   **有效範例**：「當您最近一次遇到保險公司發出『照會補件』要求時，您花了多少時間通知客戶？您當時最感到焦慮的事情是什麼？」
    *   *設計意圖*：探尋使用者最痛的痛點，用來做為我們規劃「進度追蹤功能（SOP 提醒）」的參考。

### 2. 任務達成與易用性題（量化操作阻力）
*   **有效範例**：「當您想要修改某位客戶的聯絡電話時，從點開 App 到修改完成，您覺得這個過程順暢嗎？(分值 1-5 分)」
    *   *設計意圖*：直接量化 UX 的摩擦力，驗證「翻面才能編輯」之類的設計是否真的造成了困擾。
*   **有效範例**：「在最近 5 次的語音備忘錄轉錄中，AI 幫您提取的『下次跟進時間』與您心裡預期的符合程度為何？」

### 3. 未來功能優先級排序（非虛擬發問，而是資源分配）
*   **有效範例**：「如果我們只能在下個月開發一個新功能，您會選擇以下哪一項？(單選)」
    *   [ ] 讓客戶可以在您的 LINE 機器人上預約您的空閒時間
    *   [ ] 可以將客戶的電子名片備份到 Google 雲端硬碟
    *   [ ] 自動把資料備份到個人手機行事曆中
    *   *設計意圖*：強迫使用者做選擇（Trade-off），這才是最真實的優先級需求。

## ⚠️ 三、避開「問了等於白問」的 4 大陷阱問題 (Avoid These Traps)

有些問題看似合情合理，但使用者的回答往往是出於「客套」或「對未來的錯誤預估」，這會誤導開發方向。

### 陷阱 1：虛擬未來題（問使用者「以後會不會用某功能」）
*   ❌ **白問範例**：「如果我們未來做一個 LINE 預約機器人，你會想用嗎？」
    *   *為什麼白問*：使用者通常會禮貌性地回答「會」，因為這對他們沒有損失。但真正上線時，可能因為設定複雜、操作習慣等原因，使用率接近零。
    *   *如何修改*：詢問**過去的歷史行為**：「在過去一個月內，曾有多少位客戶因為您在忙，而無法順暢跟您約定見面時間的？(請估計次數)」如果次數很多，代表這個功能才有開發價值。

### 陷阱 2：向使用者詢問「解決方案」而非「痛點」
*   ❌ **白問範例**：「您希望我們在首頁行事曆功能中增加什麼新欄位或按鈕？」
    *   *為什麼白問*：使用者不是產品設計師，他們會給出千奇百怪、甚至互相衝突的方案（例如：「我要紅色按鈕」、「我要字體超大但卡片要超小」），這會讓系統變得雜亂。
    *   *如何修改*：專注詢問**問題**：「當您在首頁瀏覽今日行程時，您最常遇到什麼不方便的地方？（例如：看不到客戶的頭像、無法一眼看出這行程約在哪裡...）」由我們來設計解決方案。

### 陷阱 3：引導性/討好性問題（迫使使用者給好評）
*   ❌ **白問範例**：「我們新設計的 3D 翻轉客戶卡片動畫是不是非常精美且實用？」
    *   *為什麼白問*：問題帶有強烈暗示，使用者很難寫出「這很浪費時間且容易滑動誤觸」的真實想法。
    *   *如何修改*：保持中立並提供負面選項：「關於客戶卡片的 3D 翻轉功能，您的使用體驗是？(複選)」
        *   [ ] 動畫效果很精緻，我很喜歡
        *   [ ] 能夠快速翻過去看備註，很方便
        *   [ ] 滑動列表時經常不小心誤觸翻面，覺得有點干擾
        *   [ ] 找編輯按鈕時卡了一下，不知道要翻面才能按

### 陷阱 4：雙重問題 (Double-Barreled Question)
*   ❌ **白問範例**：「您覺得新增客戶的對話框輸入起來快速且版面美觀嗎？」
    *   *為什麼白問*：這是兩個完全不同的概念。如果使用者覺得「輸入很快，但版面很醜」，他不知道該選「是」還是「否」。
    *   *如何修改*：拆分成兩個獨立的問題，或者專注於易用性（快速）即可。
