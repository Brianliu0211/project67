# ⌨️ 開發人員快捷指令手冊 (Developer Shortcuts)

為了提高開發效率並建立防呆機制，專案協作中定義了數個與 AI 助理對話的「快捷指令」。本手冊紀錄所有可用指令、輸入時機與 AI 對應的動作。

---

## 🧭 快捷指令快速對照表

| 指令名稱 | 輸入時機 | AI 執行的主要動作 | 備註 |
| :--- | :--- | :--- | :--- |
| **`開工`** | 開始編寫新功能前 | 確認開發者身份、展示未完任務、給予分支建議與開工 Git 指令 | 引導 GitHub Desktop 為主 |
| **`收工`** | 開發完畢、準備提交時 | 自動建立開發日誌、更新進度表、生成 Commit 訊息與合併 PR 引導 | 自動化流程整合 |
| **`讓我看看`** | 想在本地看網頁測試版時 | 自動在背景啟動預覽伺服器並提供 [http://localhost:8080](http://localhost:8080) 連結 | 便於快速測試與展示 |
| **`確認狀況`** | 迷失方向或懷疑 AI 誤解時 | **啟動安全凍結**，只讀不寫，列出已改檔案、後續變化並引導確認 | **安全手煞車** |

---

## 🛠️ 各指令詳細用法與流程

### 1. 「開工」
- **用法**：在與 AI 的對話開頭直接輸入 `開工`。
- **AI 動作**：
  1. **前置安全檢查**：自動檢查 Git 狀態，若偵測到本地有未提交變更，或前次分支未執行收工合併，將發出黃/橙色警告暫停開工程序。
  2. 詢問您是 **蘿蔔 (lobo)** 還是 **巨獸 (beast)**。
  3. 讀取並顯示 [進度.md](../進度.md) 中的當前任務供您選擇。
  4. 提供如 `feature/username-featurename` 的分支命名建議與對應 GitHub Desktop 切換步驟。

### 2. 「收工」
- **用法**：完成本地測試且編譯無誤後，輸入 `收工`。
- **AI 動作**：
  1. **智慧 Git 變更檢視與提交分流**：
     - 🟢 **純文件模式 (Fast-Track)**：若變更 **100% 僅為 `docs/` 或 `.md` 文件**，AI 提示「可免發起 PR，直接提交至 `main`」，並提供 Commit 訊息與直推 `main` 的引導。
     - 🔴 **程式碼模式 (Standard)**：若包含 `lib/` 或 `pubspec.yaml` 等程式碼，AI **強制要求**使用特徵分支，並引導 Commit、Push 至 `feature/` 分支後發起 PR。
  2. 在 `docs/03_開發日誌/` 依規範自動新增今日開發日誌。
  3. 自動在 [進度.md](../進度.md) 中勾選完成的任務。
  4. 產生可直接複製的 Git Commit 訊息與對應之推送/合併引導。

### 3. 「讓我看看」
- **用法**：輸入 `讓我看看` 請求網頁預覽。
- **AI 動作**：在背景執行預覽伺服器啟動命令，並提供點擊連結。專案人員亦可在根目錄手動雙擊 [run_local_web.bat](../../run_local_web.bat) 啟動。

### 4. 「確認狀況」
- **用法**：在開發過程中，若對目前的代碼方向有疑慮，輸入 `確認狀況`。
- **AI 動作**：進入安全防禦狀態，停止所有寫入與指令，並彙整：
  - 目前的任務進度。
  - 目前本機已修改的檔案清單與變更摘要。
  - 接下來 AI 預期要修改的內容與行為變化。
