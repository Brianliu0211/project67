# 開發行前檢查表 (Development Pre-Flight Checklist)

每次開工前（不論是蘿蔔、巨獸或 AI Agent），請先過一遍以下清單，確保環境同步與進度對齊。

---

## 🔄 1. 環境同步 (Environment Sync)

- [ ] 在 GitHub Desktop 點擊 **Fetch origin** / 執行 `git pull` 拉取最新程式碼
- [ ] 執行 `flutter pub get` 同步套件依賴
- [ ] 確認 `flutter --version` 輸出的版本與 [工具包.md](file:///c:/Users/brain/OneDrive/文件/GitHub/project67/docs/工具包.md) 記載的建議版本一致（目前建議 Flutter v3.44.x / Dart 3.12.x）
- [ ] 確認 `.env` 檔案存在且金鑰已填入（參考 `.env.example` 範本）

---

## 📍 2. 進度對齊 (Progress Alignment)

- [ ] 閱讀 [進度.md](file:///c:/Users/brain/OneDrive/文件/GitHub/project67/docs/進度.md) 確認目前的開發階段 (Phase)
- [ ] 閱讀 `docs/03_開發日誌/` 下最新的日誌，了解上次做到哪裡
- [ ] 確認本次預計要做的功能項目

---

## 🏁 3. 收工清單 (End-of-Session Checklist)

完成開發後，請執行以下步驟（或對 AI 說「**收工**」觸發自動化）：

- [ ] 確認程式碼通過 `flutter analyze` 靜態分析（無錯誤）
- [ ] 在 GitHub Desktop 提交 (Commit) 並推送 (Push)
  - Summary 與 Description 請遵循 [Git提交訊息規範.md](file:///c:/Users/brain/OneDrive/文件/GitHub/project67/docs/00_公共規格/Git提交訊息規範.md)
- [ ] （若有階段性成果）撰寫開發日誌或對 AI 說「收工」自動產出
- [ ] 確認 [進度.md](file:///c:/Users/brain/OneDrive/文件/GitHub/project67/docs/進度.md) 的勾選狀態是否需要更新
