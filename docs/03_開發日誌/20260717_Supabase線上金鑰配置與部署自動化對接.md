# 📅 2026-07-17 - 【部署自動化與安全防護】Supabase 線上金鑰配置與 Vercel 部署自動化對接

> **執行狀態**：🟢 已併入主線 (src)

## 🎯 實作成果摘要
- 解決 Vercel 線上部署版無法讀取 Supabase 金鑰而降級為「離線預覽模式」的問題。現在 Vercel 部署將能自動讀取 GitHub Actions 中配置的 Secrets，打包出具備完整 Supabase 線上資料庫對接功能的正式版本。

## 💻 技術變更明細 (Actual Technical Changes)
- **[MODIFY] `.github/workflows/deploy.yml`**：
  - 將 Dummy .env 建立步驟修改為從 GitHub Secrets 寫入實際的 `SUPABASE_URL` 與 `SUPABASE_ANON_KEY` 至 `.env` 中，讓 Flutter Web 編譯時能順利將設定封裝進 Assets 中。
  - 關鍵程式碼變更：
    ```yaml
    -      - name: Create Dummy Env File
    -        run: touch .env
    +      - name: Create Env File from Secrets
    +        run: |
    +          echo "SUPABASE_URL=${{ secrets.SUPABASE_URL }}" > .env
    +          echo "SUPABASE_ANON_KEY=${{ secrets.SUPABASE_ANON_KEY }}" >> .env
    ```

## ⚠️ 環境異動與破壞性變更 (Environment & Breaking Changes)
- **密鑰異動**：
  - 需要在 GitHub Repository Settings ➔ Secrets and variables ➔ Actions ➔ Repository secrets 中手動配置以下兩個金鑰：
    1. `SUPABASE_URL` (專案 API 網址)
    2. `SUPABASE_ANON_KEY` (專案 anon public key)
