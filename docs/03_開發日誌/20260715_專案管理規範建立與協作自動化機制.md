> **執行狀態**：🟢 已併入主線 (src)

## 🎯 實作成果摘要
- 建立完整的專案管理規範體系，包含開發行前檢查表、Git 提交訊息規範、「收工」自動化指令、金鑰範本檔案，以及專題報告暫存區，大幅提升團隊協作的一致性與開發效率。

## 💻 技術變更明細 (Actual Technical Changes)
- **[NEW] `docs/00_公共規格/開發行前檢查表.md`**：
  - 定義每次開工前的三階段檢查流程：環境同步（git pull、flutter pub get、版本確認）、進度對齊（閱讀進度表與最新日誌）、收工清單（靜態分析、提交推送、日誌撰寫）。
- **[NEW] `docs/00_公共規格/Git提交訊息規範.md`**：
  - 建立 Conventional Commits 風格的提交訊息格式指南，定義 8 種類型標籤（feat、fix、docs、style、env、refactor、test、chore），並提供混合類型的組合範例與 Description 的撰寫原則。
- **[NEW] `.env.example`**：
  - 建立金鑰範本檔案，列出 `SUPABASE_URL`、`SUPABASE_ANON_KEY`、`GEMINI_API_KEY` 三個必填環境變數，讓新加入的夥伴或新裝置能快速知道需要填入哪些金鑰。
- **[NEW] `docs/04_專題報告/README.md`**：
  - 預建專題報告暫存區資料夾，說明未來將存放系統規格書、ER 圖、流程圖、簡報與講稿等產出物。
- **[MODIFY] `.agents/AGENTS.md`**：
  - 新增「收工」自動化規則：當使用者說「收工」時，AI 必須依序執行撰寫開發日誌、更新進度表、產出 Git Commit 訊息、提示使用者推送。
  - 新增一天多份日誌的命名規則：以功能標題區分，禁止使用流水編號。
  - 新增 Git 提交訊息規範的引用規則。
