## 🎯 實作成果摘要
- 實作了全新的自訂 Toast 通知系統，以及支援 3D 翻轉動畫的客戶名片卡，並於資料庫與表單中新增綽號與照片頭像支援。

## 💻 技術變更明細 (Actual Technical Changes)
- **[MODIFY] `lib/screens/customer_management_tab.dart`**：
  - 實作 `FlippingCustomerCard` 元件，使用 `Transform` 繞 Y 軸旋轉 180 度。正面顯示 `姓名 (綽號)`、電話、信箱、標籤與頭像，反面顯示詳細備註與操作按鈕：
    ```dart
    Transform(
      transform: Matrix4.identity()
        ..setEntry(3, 2, 0.001) // perspective
        ..rotateY(transformVal),
      alignment: Alignment.center,
      child: showFrontSide ? _buildFront(...) : _buildBack(...),
    )
    ```
  - 實作 `CustomToast` 彈出式通知，分為 `success` (綠), `warning` (黃) 與 `error` (紅) 類型，支援進出場 Fade/Slide 動態效果，大螢幕預設顯示於右下角。
  - 修改 `_createCustomer`、`_updateCustomer`、`_fetchCustomers` 與 `_showCustomerForm`，全面接入自訂 Toast 通知，並支援 `nickname` 和 `avatar_url` 資料存取。
- **[MODIFY] `lib/main.dart`**：
  - 將 `ThemeData` 的 `cardTheme` 從 `CardTheme` 改為 `CardThemeData`，修復新版 Flutter SDK Web 編譯之型別錯誤。
- **[MODIFY] `.github/workflows/deploy.yml`**：
  - 移除固定舊版 Flutter SDK `3.22.2` 的設定，改為追蹤最新的 `stable` 頻道，以解決 GitHub Actions CI/CD 編譯時因 SDK 版本過舊（不支援 `CardThemeData`）導致的編譯失敗問題。
- **[MODIFY] `run_local_web.bat`**：
  - 將 `flutter_tools.snapshot` 的路徑 Commit Hash 更新為 `f94f4fc76b4d74543ed9b085bbd75341ef65de22`，以符合巨獸 (beast) 開發環境之快取目錄。
- **[DB] `supabase_schema.sql`**：
  - 在 `public.customers` 資料表中新增 `nickname TEXT` 與 `avatar_url TEXT` 欄位定義。
