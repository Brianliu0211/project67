## 💻 技術變更明細 (Actual Technical Changes)

- **[NEW] `tool/verify_toolpack.dart`**：
  - 實作自動校對核心。讀取 `docs/工具包.md` 與 `pubspec.yaml`，比對 `supabase_flutter` 等核心套件版本；調用 `puro flutter --version` 與工具包建議版本比對，若有版本不對齊則拋出警告並中斷程序。
- **[NEW] `verify_toolpack.bat`**：
  - Windows 下雙擊即可執行版本校對的批次指令檔。支援 `nopause` 參數以供自動化/CI 流程集成。
- **[MODIFY] `run_local_web.bat`**：
  - 在編譯前自動調用 `verify_toolpack.bat nopause` 進行防呆檢查。如果檢測到版本不一致，則會噴出顯著警告，並倒數 5 秒提示使用者，但不阻擋開發人員進行偵錯預覽。
- **[NEW] `docs/00_公共規格/開發人員快捷指令.md`**：
  - 新增專屬的快捷指令手冊，收錄 `開工`、`收工`、`讓我看看`、`確認狀況` 的輸入時機、流程與 AI 對應動作。
- **[MODIFY] `.agents/AGENTS.md`**：
  - 新增 **`Shortcut Guide Update`** 同步鐵律：當 AI 助理修改底層快捷指令規則時，強制自動更新人類手冊。
  - 新增 **`Obsidian Linking Specification`** 規範：禁止建立無實體檔案的抽象概念空連結、限制目錄小寫路徑（`docs/`）、限制大小寫拼寫對齊以防 GitHub 404，並優先使用相對路徑以實現 GitHub 與 Obsidian 點擊雙重相容。
  - 移除已過時的「本地網頁編譯過渡期規則」，對齊已上線的 GitHub Actions 自動化部署機制。
- **[MODIFY] `docs/00_公共規格/開發人員手冊.md`**：
  - 更新連結語法規範小節，納入新的 Obsidian 圖譜防錯標準。
  - 在開工、收工、讓我看看、自救框架段落新增對應超連結指向 [開發人員快捷指令.md](開發人員快捷指令.md)。
- **[MODIFY] `docs/工具包.md`**：
  - 修正 `supabase_flutter` 版本，從原先不一致的 `^2.16.0` 修正為 `^2.15.4`，實現文件與代碼的完美對齊。
